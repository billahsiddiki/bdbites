The Philosophy — Soft Power, Long Game

We are water, not a sword.

  • Speak well of everyone. Highlight the good in leaders, celebrities, opinion makers.
  • State facts plainly. Never sensationalize. Never attack. Never take sides.
  • Build credibility through consistency. 5 years of trust before any hard move.
  • Every post invokes emotion through CRAFT — not controversy.
  • The people decide. We provide the lens. We never tell them what to think.
  • Nobody can point fingers at us. Nobody can deny us either.
  • We are the calm in the storm. The credible default. The trusted friend.

2. Layered News — One Story, Five Angles

This is the content multiplier that makes 5 channels more powerful than 50 separate pages. One breaking story gets scored once, then automatically routed to 2-5 channels with a DIFFERENT angle per channel.

Example: "Bangladesh Bank raises interest rate by 50 basis points"
Mandate
Policy angle: what the central bank said, why, economic context
Ledger
Business impact: what this means for EMI, investment, banking sector
Rong
Skip — no entertainment angle (unless a celebrity commented)
Dheu
Viral reaction: screenshot best memes/tweets about interest rates
Jonotar
Citizen impact: "আপনার EMI কত বাড়বে? হিসাব করুন"
Example: "Shakib Al Hasan announces retirement from T20I"
Mandate
National significance: career stats, what this means for BD cricket policy
Ledger
Business angle: brand endorsement impact, cricket economy
Rong
Entertainment: career highlights carousel, "Top 5 Shakib moments"
Dheu
Viral: fan reactions, screenshots, memes, polls "best BD cricketer ever?"
Jonotar
Citizen angle: "মাগুরার ছেলে থেকে বিশ্ব তারকা — শাকিবের গল্প"

How Layering Works in the Pipeline

SCORING AGENT returns: channels: ["the_mandate", "dhaka_ledger", "dheu", "jonotar_kotha"]

FOR EACH channel in channels[]:
  → WRITER receives same article BUT different system prompt per channel
  → Mandate writer: "You write for a premium English political news brand..."
  → Ledger writer: "You write for a LinkedIn-first business analysis brand..."
  → Dheu writer: "You write viral Bangla content for mass reach..."
  → Jonotar writer: "You write for ordinary citizens affected by policy..."

RESULT: 1 article → 4 completely different posts → 4 different audiences
VOLUME MULTIPLIER: 50 scored articles/day × average 3 channels each = 150 posts/day
The Venn Diagram PrincipleEvery post should sit at the intersection of 2+ worlds. Politics + Lifestyle = "what the PM wears to summits." Sports + Life lessons = "what Shakib's comeback teaches about resilience." Business + Aspiration = "started with ৳10,000, now runs ৳10 crore company." This intersection thinking is what makes content shareworthy — it serves multiple audiences simultaneously.

4. Server Setup — Copy-Paste for a 15-Year-Old

What is a server? A computer that runs 24/7 in a data center in Germany. You rent it for $8/month. You connect to it from your laptop using a program called "Terminal" (Mac) or "PuTTY" (Windows). You type commands, it does things. That's it.

Step 1: Buy a Server (5 minutes)

  1. Open hetzner.com in your browser
  2. Click "Cloud" → "Sign Up" → Create account with your email
  3. Click "Create Server"
  4. Location: pick Falkenstein (cheapest)
  5. Image: pick Ubuntu 24.04
  6. Type: pick CX33 (€6.49/mo)
  7. Click "Create & Buy"

5. Database Setup — Copy-Paste into Supabase

What is Supabase? A database — like a giant spreadsheet that your pipeline reads and writes to. It stores every article scraped, every score, every caption, every post result. Go to supabase.com → Sign Up → New Project → Name: "bd-media" → Region: Singapore → Create. Wait 2 minutes. Click "SQL Editor" → "New Query" → paste the entire block below → click "Run".
-- BD MEDIA HQ — DATABASE SCHEMA — Run ONCE in Supabase SQL Editor

CREATE TABLE sources (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL, url TEXT NOT NULL,
  feed_type TEXT NOT NULL, language TEXT DEFAULT 'bn',
  tier INT DEFAULT 2, category TEXT,
  is_active BOOLEAN DEFAULT true,
  last_scraped_at TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE raw_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_id UUID REFERENCES sources(id),
  source_url TEXT NOT NULL, url_hash TEXT UNIQUE NOT NULL,
  title TEXT, body TEXT, image_url TEXT,
  published_at TIMESTAMPTZ, scraped_at TIMESTAMPTZ DEFAULT now(),
  language TEXT DEFAULT 'bn', processed BOOLEAN DEFAULT false
);
CREATE INDEX idx_raw_hash ON raw_items(url_hash);

CREATE TABLE content_queue (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  raw_item_id UUID REFERENCES raw_items(id),
  title TEXT NOT NULL, body TEXT,
  source_url TEXT NOT NULL, source_name TEXT, image_url TEXT,
  score INT DEFAULT 0,
  score_emotion INT, score_relevance INT, score_timeliness INT,
  score_shareability INT, score_visual INT, score_uniqueness INT, score_safety INT,
  channels TEXT[], language TEXT, format TEXT, urgency TEXT,
  hook_bangla TEXT, hook_english TEXT, deeper_layer TEXT, emotional_trigger TEXT,
  status TEXT DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_queue_status ON content_queue(status);
CREATE INDEX idx_queue_score ON content_queue(score DESC);

CREATE TABLE generated_content (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  queue_id UUID REFERENCES content_queue(id),
  channel TEXT NOT NULL, platform TEXT NOT NULL,
  headline TEXT, caption TEXT, hashtags TEXT[], cta TEXT,

8. How Photos Actually Work

What is og:image? Every news article on the internet has a hidden tag in its code: <meta property="og:image">. This is the main photo the newsroom chose for the story. When Apify scrapes an article, it grabs this URL automatically. That's your image — a real photo of a real event.
SourceHowWhen% Posts
Article og:imageApify extracts automatically during scrapeStandard news posts~85%
ScreenshotScreenshotOne API ($9/mo) → screenshot a tweet/postViral tweets, politician posts~8%
AI IllustrationFLUX 1.1 Pro via fal.ai → abstract editorial imageAbstract topics, no photo exists~4%
Text-Only CardNo image. Bold headline on brand color.Breaking news, quotes, data~3%

// This is what n8n sends to Bannerbear for every post:
POST https://api.bannerbear.com/v2/images

Body:
{
  "template": "YOUR_MANDATE_NEWS_TEMPLATE_ID",
  "modifications": [
    { "name": "bg_image", "image_url": "{{$json.image_url}}" },
    { "name": "headline", "text": "{{$json.headline}}" },
    { "name": "summary", "text": "{{$json.caption_line1}}" },
    { "name": "source", "text": "Source: {{$json.source_name}}" }
  ]
